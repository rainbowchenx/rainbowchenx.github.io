slot 插槽 - 模板动态化 - 模板结构二次加工 - 更好的复用组件 ​

  1. 默认插槽： 组件外部维护参数和结构，内部安排位置的摆放
vue
<FancyButton>
  Click me! <!-- 插槽内容 -->
</FancyButton>
<!--FancyButton中对应位置如下  -->
<button class="fancy-btn">
  <slot></slot> <!-- 插槽出口 -->
</button>
<FancyButton>
  Click me! <!-- 插槽内容 -->
</FancyButton>
<!--FancyButton中对应位置如下  -->
<button class="fancy-btn">
  <slot></slot> <!-- 插槽出口 -->
</button>

  1. 父组件传递的内容会默认传递给子组件中的 slot 位置

    面试题=> 默认插槽的实现方式？| 插槽的聚合 | 多个插槽会使何种形态 ​

    • 插槽的聚合（多个插槽）会合并单个节点渲染到 slot 对应的位置

  2. 具名插槽：在一个组件中包含多个插槽出口，以 name 来表示当前插槽的身份，从而在组件内部做到可区分

    面试题： name 仅仅是索引 => 节点是独立生成的（就是子组件中只会有父组件中 template 的内容，不会有额外的结构） ​

  • 追问：剧名插槽中的混合传参，即外部做结构的勾勒描述，内部做参数的混合 - 作用域插槽

  • 作用域插槽：父来决定插槽的结构，子来决定插槽的内容 插槽内容由子组件来决定
vue
<Hello>
  <template slot="slotContent" slot-scope="{slotProps}">
    {{ slotProps }}
  </template>
</Hello>
<!-- 子组件中,slotProps是子组件中的数据，可以以这种方式将子组件中的数据传到父组件中 -->
<slot name="slotContent" :slotProps="slotProps">
{{slotProps}}
</slot>
<Hello>
  <template slot="slotContent" slot-scope="{slotProps}">
    {{ slotProps }}
  </template>
</Hello>
<!-- 子组件中,slotProps是子组件中的数据，可以以这种方式将子组件中的数据传到父组件中 -->
<slot name="slotContent" :slotProps="slotProps">
{{slotProps}}

vue
<template>
  <div>

<script>
export default {
  data() {
    return {
      money: 100,

  },

  },

mixin - 逻辑混入（注入），数组形式 ​

  1. 应用： 逻辑相同，模板不同的情况下使用 mixin,整体结构和 vue 中 script 下的结构相同
  2. 作为逻辑的补充

面试题： mixin 的逻辑混入方式,多个 mixin 之间存在冲突怎么办？ ​

  • mixin 后者覆盖前者,mixin 和主组件冲突都以主组件为主，生命周期的混入方式，优先 mixin 生命周期加载，按照顺序，最后是主组件 mixin 在引用顺序基础上加载生命周期,都在本体之前

为什么组件需要注册：方便进行条件查找渲染，但是在 vue3 中可以直接用 mixin 本质是注入型的逻辑块，仅仅是逻辑，相当于深拷贝

  • 替换型策略有 props、methods、inject、computed，就是将新的同名参数替代旧的参数
  • 合并型策略是 data, 通过 set 方法进行合并和重新赋值
  • 队列型策略有生命周期函数和 watch，原理是将函数存入一个数组，然后正序遍历依次执行
  • 叠加型有 component、directives、filters，通过原型链进行层层的叠加

继承拓展 extend - 逻辑上的共同拓展 ​

使用 extends：直接加入，结构和 mixin 一样 应用：权限校验

  1. 核心逻辑的功能继承,extends 只能一个 extends 部分的逻辑是最先执行 合并策略：变量补充维度 => 额外补充 生命周期： 无论业务代码还是 mixin 都在 extends 之后

  2. 全局的拓展 extend & plugin

vue.extend() vue.use(vuex)

上述这些都是在 vue2 中的，在 vue3 中都没有了，直接导出函数，导入函数进行使用就行 vue3 中只需要构造出一个钩子就行，摒弃了 vue2 中本身的语法等，以函数为主
